# Surfaces And Routing

## Brand Hierarchy

Stria Systems is the company surface for accountability infrastructure. Trace is the first product surface and application housed within Stria Systems.

## Public Surfaces

- `/` presents Stria Systems and the operational record layer thesis.
- `/trace` presents Trace as evidence infrastructure and operational intelligence for AI agents.
- `/architecture` presents the data plane, control plane, and trust plane architecture.
- `/trace/documentation` presents developer docs for the local Trace prototype.
- `/docs` redirects to `/trace/documentation`.
- `/demo` captures demo requests before product onboarding.

## Deferred Authenticated Surface

- `/app/trace` is intentionally not part of the public website pass.

The product application should remain under the Stria account and organization model. Trace-specific navigation should begin only after a user has entered an authenticated product surface.

## Future Domain Mapping

- `striasystems.com` maps to `/`.
- `striasystems.com/trace` maps to the product page.
- `striasystems.com/architecture` maps to the architecture overview.
- `docs.striasystems.com` can become a dedicated documentation app when docs mature beyond the current developer landing page.
- `app.striasystems.com/trace` can map to the authenticated Trace app when tenant onboarding exists.

## Product Principle

The public website earns trust and routes qualified enterprise users into the software. The software should not carry marketing responsibilities; it should operate as an evidence console for work already underway.
